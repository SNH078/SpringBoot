# Maven Lifecycle  
 sequence of steps Maven follows to build your project  
 ```clean → compile → test → package → install → deploy```  
🔹 clean  - Deletes old files
🔹 compile - Compiles Java code
🔹 test    -  Runs tests
🔹 package  - Creates .jar file
🔹 install  - Stores in local repo 

 ## 🔄 How Maven works 
```
You write pom.xml                   (👉 This tells Maven:“I need this library”)
        ↓ 
Maven reads dependencies            (GAV)
        ↓
Checks local repo                  (👉 If dependency exists:✅ Use it , 👉 If not:⬇️ Download it )
        ↓
Downloads from internet(if needed)  👉(Internet → Maven Central Repository)
        ↓
Stores locally (in .m2)             👉 Next time → no download needed
        ↓
Add to project                      👉 Maven: Links JARs to your project  ,  Makes classes available
        ↓
Builds project (maven lifecycle runs)
```

# GAV   
GroupId + ArtifactId + Version  
👉 unique identifier of a project or dependency in Maven
```
<groupId>org.springframework.boot</groupId>
<artifactId>spring-boot-starter-web</artifactId>
<version>3.2.0</version>
```
| Field      | Meaning                |
| ---------- | ---------------------- |
| GroupId    | Organization / company |
| ArtifactId | Project name           |
| Version    | Version of project     |

👉 Maven uses GAV to: Find the library , Download it
