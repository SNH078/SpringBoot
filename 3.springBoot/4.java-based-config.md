# java-Based Configuration    
configuring Spring using Java classes instead of XML  

Java-based config has two approaches: automatic (@Component) and manual (@Bean in @Configuration).  
<img width="346" height="697" alt="image" src="https://github.com/user-attachments/assets/fc3fd3b3-312a-41fe-ad30-f6cdea4d431f" />

👉 In Spring Boot:  
Use automatic mostly ✅  
Use manual only when needed

# @Configuration   
marks a class as a configuration class where you define beans    
<img width="285" height="418" alt="image" src="https://github.com/user-attachments/assets/6911d027-5308-4a08-82ac-6cfd63336709" />
👉 This tells Spring:  
“This class contains bean definitions”

# AnnotationConfigApplicationContext 
It is a Spring container used to load configuration from Java-based config (@Configuration) or annotations    
It:   
Starts Spring  
Creates beans  
Stores them  
Manages them
