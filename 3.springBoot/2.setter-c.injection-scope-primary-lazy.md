# Setter Injection  
Create object first, then set dependency later using a setter method   

<img width="334" height="456" alt="image" src="https://github.com/user-attachments/assets/35197d72-fc16-4261-8498-1f1c9210db15" />
<img width="264" height="300" alt="image" src="https://github.com/user-attachments/assets/489dce9d-6128-4968-a24f-1ab02290f43f" />

### Key Features  

Dependency can be changed later  
Optional dependencies possible  
More flexible than constructor injection

### 🔴 Drawbacks  

Object can be in incomplete state (if setter not called)  
Not immutable  
Harder to ensure required dependencies

---

# constructor Injection  
injecting dependencies through the constructor at the time of object creation.

 <img width="404" height="441" alt="image" src="https://github.com/user-attachments/assets/c6c2e237-37f5-4ef0-8b94-e482e2e40b06" />

* use const I : when dependency is mandatory (compulsory)  (✔ Object cannot work without it)
* use setter I : when dependency is optional (✔ Object can exist without it (initially))

# Component Scan  
Spring automatically finds classes annotated with @Component and creates beans  

🔥 What it does

👉 Spring searches for:  
@Component  
@Service  
@Repository  
@Controller

👉 Then :   
Creates objects (beans)  
Adds them to container

# Lazy Initialization (@Lazy)  
Bean is created only when it is needed (used), not at startup

By default, Spring does  
👉 Eager Initialization   :   All beans are created at application startup

<img width="291" height="617" alt="image" src="https://github.com/user-attachments/assets/e0239d6a-d109-4699-b14d-e441b3e79489" />

🟢 When to use Lazy  
Heavy objects (DB connections, large configs  
Improve startup time  
Rarely used beans
